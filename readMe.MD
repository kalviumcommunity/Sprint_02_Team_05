## Key Folders Explained

| Folder | Role |
|--------|------|
| `lib/` | All Dart application code ‚Äî screens, widgets, models, services. `main.dart` is the entry point. |
| `lib/screens/` | One widget per screen. Currently: welcome, responsive home, login, signup, dashboard. |
| `lib/services/` | Business logic isolated from UI ‚Äî `auth_service.dart` and `firestore_service.dart`. |
| `android/` | Gradle build scripts, `AndroidManifest.xml`, `google-services.json` for Firebase. |
| `ios/` | Xcode project files, `Info.plist` for permissions and metadata. |
| `test/` | Widget and unit tests using Flutter's testing framework. |
| `pubspec.yaml` | Manages dependencies (`firebase_core`, `firebase_auth`, `cloud_firestore`), assets, and fonts. |

## Reflection

### Why is it important to understand each folder's role?

- **Efficiency** ‚Äî knowing where files belong saves time when adding features or debugging
- **Consistency** ‚Äî the team follows the same conventions, making PRs predictable
- **Platform isolation** ‚Äî `android/`, `ios/`, `web/` folders keep platform config separate from shared Dart code in `lib/`

### How does a clean structure help in a team environment?

- **Parallel development** ‚Äî UI developer works in `screens/`, backend developer in `services/`, with minimal merge conflicts
- **Code reviews** ‚Äî changes are scoped to specific folders, making reviews faster
- **Onboarding** ‚Äî new members understand the codebase structure immediately from folder names
- **Scalability** ‚Äî adding a new feature follows the same pattern: add a screen, reuse widgets, call services
